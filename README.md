# terraform-k8s-argus

A Terraform module that bootstraps a production-ready Kubernetes observability stack across AWS (EKS), GCP (GKE), and Azure (AKS).

**Stack**
- [kube-prometheus-stack](https://github.com/prometheus-community/helm-charts/tree/main/charts/kube-prometheus-stack) — Prometheus, Alertmanager, and Grafana
- [Loki](https://grafana.com/oss/loki/) — log aggregation (optional)
- [OpenTelemetry Collector](https://opentelemetry.io/docs/collector/) — metrics and traces (optional)

Pairs with [Ithildin](https://ithildin.onrender.com/) to load SLO-driven alerting rules directly into Prometheus at deploy time.

---

## Usage

### AWS (EKS)

```hcl
module "argus" {
  source = "github.com/balakrishnan-vasudevan/terraform-k8s-argus//aws"

  grafana_admin_password     = var.grafana_password
  alertmanager_slack_webhook = var.slack_webhook

  # Optional: IRSA role for Prometheus to access AWS services
  irsa_role_arn = var.prometheus_irsa_role_arn

  # Optional: load SLO rules generated by Ithildin
  ithildin_rules = file("${path.module}/slo-rules.yaml")
}
```

Configure your providers:

```hcl
data "aws_eks_cluster" "this" { name = "my-cluster" }
data "aws_eks_cluster_auth" "this" { name = "my-cluster" }

provider "kubernetes" {
  host                   = data.aws_eks_cluster.this.endpoint
  cluster_ca_certificate = base64decode(data.aws_eks_cluster.this.certificate_authority[0].data)
  token                  = data.aws_eks_cluster_auth.this.token
}

provider "helm" {
  kubernetes {
    host                   = data.aws_eks_cluster.this.endpoint
    cluster_ca_certificate = base64decode(data.aws_eks_cluster.this.certificate_authority[0].data)
    token                  = data.aws_eks_cluster_auth.this.token
  }
}
```

### GCP (GKE)

```hcl
module "argus" {
  source = "github.com/balakrishnan-vasudevan/terraform-k8s-argus//gcp"

  grafana_admin_password = var.grafana_password

  # Optional: Workload Identity for Prometheus
  workload_identity_sa = "prometheus@my-project.iam.gserviceaccount.com"

  ithildin_rules = file("${path.module}/slo-rules.yaml")
}
```

Configure your providers:

```hcl
data "google_container_cluster" "this" { name = "my-cluster"; location = "us-central1" }
data "google_client_config" "this" {}

provider "kubernetes" {
  host                   = "https://${data.google_container_cluster.this.endpoint}"
  cluster_ca_certificate = base64decode(data.google_container_cluster.this.master_auth[0].cluster_ca_certificate)
  token                  = data.google_client_config.this.access_token
}

provider "helm" {
  kubernetes {
    host                   = "https://${data.google_container_cluster.this.endpoint}"
    cluster_ca_certificate = base64decode(data.google_container_cluster.this.master_auth[0].cluster_ca_certificate)
    token                  = data.google_client_config.this.access_token
  }
}
```

### Azure (AKS)

```hcl
module "argus" {
  source = "github.com/balakrishnan-vasudevan/terraform-k8s-argus//azure"

  grafana_admin_password = var.grafana_password

  # Optional: Managed Identity for Prometheus
  managed_identity_client_id = "00000000-0000-0000-0000-000000000000"

  ithildin_rules = file("${path.module}/slo-rules.yaml")
}
```

Configure your providers:

```hcl
data "azurerm_kubernetes_cluster" "this" { name = "my-cluster"; resource_group_name = "my-rg" }

provider "kubernetes" {
  host                   = data.azurerm_kubernetes_cluster.this.kube_config[0].host
  cluster_ca_certificate = base64decode(data.azurerm_kubernetes_cluster.this.kube_config[0].cluster_ca_certificate)
  client_certificate     = base64decode(data.azurerm_kubernetes_cluster.this.kube_config[0].client_certificate)
  client_key             = base64decode(data.azurerm_kubernetes_cluster.this.kube_config[0].client_key)
}

provider "helm" {
  kubernetes {
    host                   = data.azurerm_kubernetes_cluster.this.kube_config[0].host
    cluster_ca_certificate = base64decode(data.azurerm_kubernetes_cluster.this.kube_config[0].cluster_ca_certificate)
    client_certificate     = base64decode(data.azurerm_kubernetes_cluster.this.kube_config[0].client_certificate)
    client_key             = base64decode(data.azurerm_kubernetes_cluster.this.kube_config[0].client_key)
  }
}
```

## Secrets management

Never hardcode passwords or webhook URLs in Terraform. The examples show the recommended approach for each cloud — fetch secrets at runtime from your cloud's secret store. The values are passed to the module as sensitive variables and never written to version control.

**AWS — Secrets Manager**
```hcl
data "aws_secretsmanager_secret_version" "grafana" {
  secret_id = "observability/grafana-admin-password"
}

module "argus" {
  source                 = "github.com/balakrishnan-vasudevan/terraform-k8s-argus//aws"
  grafana_admin_password = data.aws_secretsmanager_secret_version.grafana.secret_string
}
```

**GCP — Secret Manager**
```hcl
data "google_secret_manager_secret_version" "grafana" {
  secret = "grafana-admin-password"
}

module "argus" {
  source                 = "github.com/balakrishnan-vasudevan/terraform-k8s-argus//gcp"
  grafana_admin_password = data.google_secret_manager_secret_version.grafana.secret_data
}
```

**Azure — Key Vault**
```hcl
data "azurerm_key_vault_secret" "grafana" {
  name         = "grafana-admin-password"
  key_vault_id = data.azurerm_key_vault.this.id
}

module "argus" {
  source                 = "github.com/balakrishnan-vasudevan/terraform-k8s-argus//azure"
  grafana_admin_password = data.azurerm_key_vault_secret.grafana.value
}
```

Alternatively, pass the value via an environment variable without storing it anywhere in code:
```bash
export TF_VAR_grafana_admin_password="yourpassword"
terraform apply
```

> **Note:** `sensitive = true` on Terraform variables redacts values from CLI output but does **not** protect them in state. Use an encrypted backend (S3 with SSE-KMS, GCS with CMEK, Azure Blob with customer-managed keys) and restrict state read access to your CI role.

## Ithildin integration

[Ithildin](https://ithildin.onrender.com/) generates multi-burn-rate SLO alerting rules for Prometheus. Give it a service name and an SLO target and it produces Prometheus recording rules, multi-burn-rate alerting rules, and a Grafana dashboard. Pass the rules output directly to Argus:

```hcl
module "argus" {
  source = "github.com/balakrishnan-vasudevan/terraform-k8s-argus//aws"

  grafana_admin_password = var.grafana_password
  ithildin_rules         = file("${path.module}/slo-rules.yaml")
}
```

Argus loads the rules into Prometheus via `additionalPrometheusRulesMap`, so they take effect immediately without any manual steps.

## Variables

| Name | Description | Default |
|---|---|---|
| `namespace` | Kubernetes namespace for the stack | `"observability"` |
| `storage_class` | Storage class for PVs (auto-set by cloud wrapper) | — |
| `prometheus_retention` | Prometheus data retention | `"15d"` |
| `prometheus_storage_size` | PV size for Prometheus | `"50Gi"` |
| `alertmanager_storage_size` | PV size for Alertmanager | `"10Gi"` |
| `grafana_enabled` | Deploy Grafana | `true` |
| `grafana_admin_password` | Grafana admin password | required |
| `grafana_storage_size` | PV size for Grafana | `"10Gi"` |
| `loki_enabled` | Deploy Loki for log aggregation | `false` |
| `loki_storage_size` | PV size for Loki | `"50Gi"` |
| `otel_enabled` | Deploy OpenTelemetry Collector | `false` |
| `ingress_enabled` | Expose Grafana via ingress | `false` |
| `ingress_class` | Ingress class (auto-set by cloud wrapper) | `"nginx"` |
| `grafana_ingress_host` | Hostname for Grafana ingress | `""` |
| `alertmanager_slack_webhook` | Slack webhook URL for alerts | `""` |
| `alertmanager_pagerduty_key` | PagerDuty routing key for alerts | `""` |
| `ithildin_rules` | Prometheus rules YAML from Ithildin | `""` |
